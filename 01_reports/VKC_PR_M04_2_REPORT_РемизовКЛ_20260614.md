#### Этап 1. Получение токена аутентификации

**Действие:**  
Отправил POST-запрос к эндпоинту Identity API (`https://cloud.vk.com/identity/v3/auth/tokens`) с JSON-телом, содержащим логин, пароль и ID проекта. Пароль был введён открыто в команде (для учебных целей), в реальной среде следует использовать переменные окружения.

```bash
curl -i -X POST https://cloud.vk.com/identity/v3/auth/tokens \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["password"],
        "password": {
          "user": {
            "name": "remizov.kirill2005@yandex.ru",
            "password": "#*Woron210105#*",
            "domain": {"name": "users"}

      },
      "scope": {
        "project": {

      }

  }'
```

Сервер вернул HTTP/2 201, в заголовке `X-Subject-Token` был получен токен. Токен был сохранён в переменную окружения `$TOKEN` для использования в последующих запросах.

#### Этап 2. Получение информации о конфигурациях (flavors, сети, образы)

**Действие:**  
Используя полученный токен, отправил GET-запросы к API Compute (`/flavors/detail`), Network (`/v2.0/networks`) и Image (`/v2/images`), чтобы определить идентификаторы необходимых ресурсов для создания ВМ. Для удобства вывода использовал `jq`.

```bash
# 1. Список flavors (типов ВМ) с фильтрацией по размеру диска > 0
curl -s -X GET "https://infra.mail.ru:8774/v2.1/5aa354d05172439abcc5bfc3f682da49/flavors/detail" \
  -H "X-Auth-Token: $TOKEN" | jq '.flavors[] | {name: .name, id: .id, disk: .disk} | select(.disk > 0)'

# 2. Список сетей
curl -s -X GET "https://infra.mail.ru:9696/v2.0/networks" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" | jq '.networks[] | {id, name}'

# 3. Список образов
curl -s -X GET "https://infra.mail.ru:9292/v2/images" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" | jq '.images[] | {id, name}'
```

**Результат:**  
- Выбрал flavor `Basic-1-2-20` (id `25ae869c-be29-4840-8e12-99e046d2dbd4`) – 1 vCPU, 2 ГБ RAM, 20 ГБ диск.  
- Сеть `internet` (id `ec8c610e-6387-447e-83d2-d2c541e88164`).  
- Образ `ubuntu-24-202602051634.gite7a38aaf` (id `b812255f-7b5f-4fd6-bad4-8b24265dc076`).

#### Этап 3. Создание ВМ через API

**Действие:**  
Отправил POST-запрос к API Compute (`/servers`) с JSON-телом, содержащим имя ВМ, ссылки на образ, flavor, сеть и имя SSH-ключа.

```bash
curl -s -X POST "https://infra.mail.ru:8774/v2.1/5aa354d05172439abcc5bfc3f682da49/servers" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "server": {
      "name": "api-created-vm",
      "imageRef": "b812255f-7b5f-4fd6-bad4-8b24265dc076",
      "flavorRef": "25ae869c-be29-4840-8e12-99e046d2dbd4",
      "networks": [{"uuid": "ec8c610e-6387-447e-83d2-d2c541e88164"}],
      "key_name": "my-first-vm-ie9Ev9Cc"
    }
  }'
```

**Результат:**  
Сервер вернул JSON с информацией о создаваемой ВМ: присвоен ID `8c3f3134-560c-4558-9060-c3c6731792a8`, сгенерирован временный пароль администратора `adminPass` (не использовался, так как доступ по SSH-ключу). Статус ВМ на момент ответа – `BUILD`.

#### Этап 4. Проверка списка ВМ

**Действие:**  
Выполнил GET-запрос к списку серверов до и после создания, чтобы убедиться, что новая ВМ появилась.

```bash
# До создания (список пуст)
curl -s -X GET "https://infra.mail.ru:8774/v2.1/5aa354d05172439abcc5bfc3f682da49/servers" \
  -H "X-Auth-Token: $TOKEN" | jq '.servers'

# После создания (появилась api-created-vm)
curl -s -X GET "https://infra.mail.ru:8774/v2.1/5aa354d05172439abcc5bfc3f682da49/servers" \
  -H "X-Auth-Token: $TOKEN" | jq '.servers'
```

**Результат:**  
В списке серверов появилась запись с именем `api-created-vm` и соответствующим ID.

- **Что можно улучшить:**  
  В следующий раз стоит автоматизировать процесс: написать небольшой bash-скрипт, который получает токен, подставляет его в переменные, выполняет создание ВМ и проверяет статус. Также полезно изучить, как обрабатывать пагинацию в ответах API (при большом количестве ресурсов).
