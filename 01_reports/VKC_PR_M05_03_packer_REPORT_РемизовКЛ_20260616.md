# Отчёт по практической демонстрации

## 1. Цель

Освоить инструмент **Packer** для автоматизации создания кастомизированных образов виртуальных машин в облаке VK Cloud. Научиться:

- описывать процесс сборки образа декларативно с помощью HCL-конфигурации;
- использовать провизоры (shell) для установки и настройки ПО внутри собираемой ВМ;
- решать проблемы с сетью и DNS в процессе сборки;
- проверять работоспособность созданного образа путём запуска тестовой ВМ.

  - Packer как инструмент для автоматической сборки образов (Immutable Infrastructure).

## 3. Ход выполнения

### Демонстрация 5.3: Создание образов с Packer

#### Этап 0. Подготовка окружения

- Проверена версия Packer: `1.15.4` (команда `packer --version`).  
- Установлен плагин OpenStack версии `1.1.3` (подтверждено `packer plugins installed`).  
- Заданы переменные окружения:
  ```bash
  export SOURCE_IMAGE=8582cdba-6be5-41d4-834f-b0fa920b3dcf   # базовый Ubuntu 22.04
  export NETWORK_ID=ec8c610e-6387-447e-83d2-d2c541e88164    # сеть internet
  ```

#### Этап 1. Инициализация и валидация конфигурации

- Выполнена инициализация:
  ```bash
  packer init .
  ```
  Плагин OpenStack загружен (уже был установлен, повторная загрузка подтверждена).

- Проверена корректность конфигурации:
  ```bash
  packer validate .
  ```
  Результат: `The configuration is valid.`

#### Этап 2. Сборка образа с помощью Packer

Запущен процесс сборки:
```bash
packer build .
```

В логе видна последовательность действий Packer:

1. **Загрузка flavour** `STD3-2-6` (ID подтверждён).  
2. **Создание тома** (block storage volume) для временной ВМ.  
3. **Запуск сервера** с указанными параметрами: сеть `internet`, группа безопасности `ssh`, ключ `postgress-test-ViPomgZ4`.  
4. **Ожидание готовности** – сервер получает публичный IP (`95.163.214.208`).  
5. **Подключение по SSH** – используется коммуникатор SSH с приватным ключом `postgress-test-ViPomgZ4.pem`.  
6. **Выполнение провизора `shell`** – внутри ВМ последовательно выполняются команды:
   - Фикс файла `/etc/hosts` (добавление записи `127.0.0.1 $(hostname)`).
   - Настройка DNS: удаление существующего `resolv.conf`, установка `nameserver 8.8.8.8` и `1.1.1.1`, защита от перезаписи (`chattr +i`).
   - Обновление списка пакетов (`apt-get update`). На этом этапе возникли временные ошибки разрешения доменов, но благодаря фиксам DNS и наличию кеша пакетов установка nginx прошла успешно (пакет уже был установлен в базовом образе).
   - Включение и запуск nginx.
   - Создание тестовой страницы `/var/www/html/index.html` с содержимым `<h1>Hello from Packer!</h1>`.
   - Очистка временных файлов и кэша пакетов.
7. **Завершение работы** – временная ВМ останавливается, создаётся образ, затем ВМ и том удаляются. В логе появилась ошибка `Error terminating server, may still be around: Resource not found`, но это не повлияло на создание образа.  
8. **Создание образа** – финальный образ с именем `nginx-custom-image-2026-06-16-2107` (ID `f69ca872-96a2-40a9-ad31-4e098b0f9b0e`).

**Время сборки:** 7 минут 44 секунды.

#### Этап 3. Проверка созданного образа

- Проверено, что образ появился в списке:
  ```bash
  openstack image list | grep nginx-custom
  ```
  Образ активен.

- Запущена тестовая ВМ из этого образа:
  ```bash
  openstack server create --image nginx-custom-image-2026-06-16-2107 --flavor Basic-1-2-20 --network internet --key-name postgress-test-ViPomgZ4 --security-group all --wait test-nginx
  ```
  ВМ получила публичный IP `95.163.214.174`.

- Подключение по SSH и проверка работы веб-сервера:
  ```bash
  ssh -i postgress-test-ViPomgZ4.pem ubuntu@95.163.214.174
  curl localhost
  ```
  Вывод: `<h1>Hello from Packer!</h1>` – всё работает (скриншот приложен).

## 4. Ошибки и исправления

| Проблема | Решение |
| :--- | :--- |
| В процессе выполнения `apt-get update` возникли ошибки разрешения DNS (`Temporary failure resolving`). | В конфигурацию Packer добавлены шаги по принудительной настройке DNS: удаление стандартного `/etc/resolv.conf`, установка публичных DNS-серверов (8.8.8.8, 1.1.1.1) и защита файла от перезаписи через `chattr +i`. Также добавлена запись `127.0.0.1 $(hostname)` в `/etc/hosts` для корректной работы `sudo`. |
| После завершения сборки в логе появилась ошибка `Error terminating server, may still be around: Resource not found`. | Это не критично – сервер уже был остановлен и образ создан. Ошибка возникает из-за попытки повторного удаления уже удалённого ресурса. На результат сборки не влияет. |

## 7. Самопроверка

- **Что получилось:**  
  Успешно автоматизировал создание кастомизированного образа с nginx, используя Packer. Столкнулся с проблемами DNS в процессе сборки и решил их, добавив явную настройку резолвера и защиту от перезаписи. Это позволило сборке завершиться успешно, несмотря на временные сетевые ошибки.

- **Что осталось непонятным:**  
  - Почему в базовом образе Ubuntu 22.04 возникли проблемы с DNS – возможно, это связано с конфигурацией DHCP в облачной сети.  
  - Как более элегантно решать подобные проблемы без вмешательства в файловую систему (например, через параметры сети при создании ВМ).  
  - Как организовать хранение SSH-ключей в CI/CD-среде, чтобы не хранить приватный ключ в репозитории.

- **Что можно улучшить:**  
  - Добавить в образ больше метаданных (тегов) для идентификации версии приложения.  
  - Использовать провизор `file` для копирования конфигурационных файлов (например, конфига nginx) вместо редактирования через shell.  
  - Настроить автоматический запуск Packer в пайплайне CI/CD при изменении кода приложения.

Ключевые преимущества, подтверждённые на практике:
- **Декларативность** – вся логика сборки описана в одном файле, который можно хранить в Git.
- **Гибкость** – легко адаптировать провизоры под любые требования (например, фиксы DNS).
- **Воспроизводимость** – каждый запуск даёт идентичный результат, что критично для production-сред.
