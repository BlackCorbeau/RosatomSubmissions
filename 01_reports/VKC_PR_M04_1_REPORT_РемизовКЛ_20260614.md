# Отчет по практической демонстрации

## 1. Цель

Освоить управление облачными ресурсами VK Cloud с помощью **OpenStack CLI**.  
Научиться выполнять основные операции: аутентификация через RC‑файл, создание и удаление виртуальных машин, создание снапшотов (образов) из работающей ВМ.  
Закрепить понимание преимуществ командной строки перед веб‑консолью для автоматизации и повторяемости действий.

## 2. Что было прочитано перед выполнением

- **Ключевые понятия:**  
  - RC‑файл (OpenStack RC File) и переменные окружения

#### Этап 2. Создание виртуальной машины через CLI

**Действие:**  
Создал ВМ с именем `cli-advanced-vm`, используя готовые идентификаторы образа, сети, ключа и группы безопасности.

```bash
openstack server create \
  --flavor df3c499a-044f-41d2-8612-d303adc613cc \
  --image "ubuntu-22-202602051629.gite7a38aaf" \
  --key-name my-first-vm-ie9Ev9Cc \
  --network "ec8c610e-6387-447e-83d2-d2c541e88164" \
  --security-group ssh \
  cli-advanced-vm
```

**Результат:**  
ВМ перешла в статус `BUILD`, а затем (после завершения создания) – в `ACTIVE`.  
В выводе команды отобразились параметры: `id`, `flavor`, `image`, `security_groups`, `adminPass` (не использовался, так как доступ по SSH‑ключу).

#### Этап 3. Создание снапшота (образа) из работающей ВМ

**Действие:**  
Для получения консистентного образа сначала остановил ВМ, затем создал образ `my-custom-image`, дождался завершения (`--wait`), проверил список образов и снова запустил ВМ.

```bash
openstack server stop cli-advanced-vm

openstack server image create \
  --name my-custom-image \
  --wait \
  cli-advanced-vm

openstack image list | grep my-custom-image

openstack server start cli-advanced-vm
```

**Результат:**  
- ВМ остановлена (в логах видно, что перед созданием образа ВМ находилась в статусе `SHUTOFF`).  
- Образ `my-custom-image` успешно создан со статусом `active`, размер 10 ГБ, тип `snapshot`.  
- В списке образов появилась запись с ID `84104fb4-7ee8-4472-aa1d-ecfe0b7b16d6`.  
- ВМ запущена обратно.

## 7. Самопроверка

- **Что получилось:**  
  Уверенно выполнил все команды OpenStack CLI: аутентификация, создание ВМ, управление состоянием (stop/start), создание образа. Понял, как связаны идентификаторы (flavor, image, network) и как их узнать через `openstack flavor list`, `openstack image list`, `openstack network list`. Создание снапшота прошло без проблем – образ появился и готов к использованию.

- **Что осталось непонятным:**  
  В выводе `openstack server create` присутствует поле `adminPass`. Не вполне ясно, при каких условиях оно используется (вероятно, для образов с парольной аутентификацией). В данном случае доступ осуществляется по SSH‑ключу, поэтому этот пароль не применяется.

- **Что можно улучшить:**  
  В будущем стоит попробовать создать образ без остановки ВМ (с опцией `--force`, если она поддерживается) – для сценариев, где недоступен downtime. Также полезно освоить команду `openstack server add security group` для динамического изменения политик безопасности.

## 8. Краткий вывод

Практическая работа показала, что **OpenStack CLI** является мощным и гибким инструментом для управления облачной инфраструктурой VK Cloud. В отличие от веб‑консоли, командная строка позволяет выполнять операции быстрее, легко встраивать их в скрипты и системы автоматизации (Ansible, CI/CD). Создание снапшота через CLI (`openstack server image create`) даёт возможность быстро создавать «золотые образы» для последующего развёртывания идентичных ВМ.

**Ключевой вывод:**  
Освоение CLI – обязательный шаг на пути от ручного администрирования к профессиональной автоматизации и практике Infrastructure as Code (IaC).
